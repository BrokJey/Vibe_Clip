# Настройка онлайн доступа для Vibe Clip

## Шаг 1: Настройка Backend

Backend уже настроен для работы в онлайн режиме:
- ✅ Сервер слушает на `0.0.0.0:8000` (все интерфейсы)
- ✅ CORS настроен для разрешения всех источников
- ✅ Порт 8000 открыт для внешних подключений

## Шаг 2: Получение IP адреса

### Windows:
```powershell
ipconfig
```
Найдите **IPv4 адрес** в разделе вашего сетевого адаптера (например, `192.168.1.100`)

### Mac/Linux:
```bash
ifconfig | grep "inet "
```
или
```bash
hostname -I
```

### Важно:
- Используйте **локальный IP** (192.168.x.x или 10.x.x.x) для тестирования в локальной сети
- Для доступа из интернета нужен **публичный IP** и настройка роутера (port forwarding)

## Шаг 3: Настройка Android приложения

1. Откройте файл: `VibeClip_Frontend/app/src/main/java/com/example/vibeclip_frontend/data/RetrofitClient.kt`

2. Замените `YOUR_IP_ADDRESS` на ваш IP адрес:
```kotlin
private const val BASE_URL = "http://192.168.1.100:8000/api/v1/"
```

3. Пересоберите приложение

## Шаг 4: Настройка Firewall

### Windows:
1. Откройте "Брандмауэр Защитника Windows"
2. Нажмите "Дополнительные параметры"
3. Выберите "Правила для входящих подключений" → "Создать правило"
4. Выберите "Порт" → "TCP" → "Определенные локальные порты: 8000"
5. Разрешите подключение
6. Примените для всех профилей

### Mac:
```bash
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /path/to/java
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblockapp /path/to/java
```

### Linux (ufw):
```bash
sudo ufw allow 8000/tcp
```

## Шаг 5: Проверка подключения

### С компьютера:
```bash
curl http://YOUR_IP_ADDRESS:8000/api/v1/auth/login
```

Должен вернуться ответ (даже если ошибка, главное что сервер отвечает).

### С Android устройства:
1. Убедитесь, что устройство подключено к той же сети Wi-Fi
2. Откройте браузер на устройстве
3. Перейдите по адресу: `http://YOUR_IP_ADDRESS:8000/api/v1/auth/login`

## Шаг 6: Запуск Backend

```bash
cd Vibe_Clip
mvn spring-boot:run
```

Убедитесь, что видите в логах:
```
Tomcat started on port(s): 8000 (http) with context path ''
```

## Доступ из интернета (опционально)

Для доступа из интернета (не только локальная сеть):

1. **Настройте Port Forwarding на роутере:**
   - Порт: 8000
   - Протокол: TCP
   - Внутренний IP: IP вашего компьютера
   - Внутренний порт: 8000

2. **Получите публичный IP:**
   - Перейдите на https://whatismyipaddress.com/
   - Используйте этот IP в Android приложении

3. **Важно для безопасности:**
   - Используйте HTTPS (настройте SSL сертификат)
   - Ограничьте доступ по IP адресам
   - Используйте сильные пароли

## Troubleshooting

### Проблема: "Connection refused"
- ✅ Проверьте, что backend запущен
- ✅ Проверьте firewall настройки
- ✅ Убедитесь, что IP адрес правильный
- ✅ Проверьте, что устройство в той же сети

### Проблема: "Timeout"
- ✅ Проверьте, что порт 8000 открыт в firewall
- ✅ Проверьте настройки роутера (если используете port forwarding)
- ✅ Убедитесь, что backend слушает на 0.0.0.0, а не localhost

### Проблема: "CORS error"
- ✅ Убедитесь, что в `SecurityConfig.java` установлен `setAllowedOrigins("*")`
- ✅ Перезапустите backend после изменений

## Безопасность

⚠️ **Важно для production:**
1. Используйте HTTPS вместо HTTP
2. Настройте правильную CORS политику (не разрешайте все источники)
3. Используйте сильные JWT секреты
4. Настройте rate limiting
5. Используйте reverse proxy (nginx) для дополнительной защиты
